Square Terminal — What You Need to Know

📄 Receipts after payment

Yes — Square Terminal prints a receipt automatically after every card payment if a receipt printer is paired to the device. The Square Terminal (hardware) has a built-in receipt printer; it prints immediately once the charge is captured. You can also configure Square to prompt the customer: Print / Email / No receipt. The app's own "Print this ticket" button (on the order detail page) is a separate browser print — that's for your kitchen/bar copy, not the card receipt.

🧪 Testing without a card reader

Square provides a Sandbox environment — no real reader needed:

  1. Sign up free at developer.squareup.com
  2. Create an application and grab your Sandbox Access Token
  3. Use the Square Terminal API with "action": "CHECKOUT" — in sandbox mode a virtual terminal completes the payment without hardware
  4. Test card numbers: 4111 1111 1111 1111 (Visa), any future expiry, any CVV
  5. Switch to Production tokens + a real Device ID when you buy the hardware

Square Terminal hardware starts at ~$299 (Terminal) or ~$49 (Reader S2 for phone/tablet). Docs: developer.squareup.com/docs/terminal-api

How Stripe Terminal Works (once wired up)

  1. Staff taps Charge & Print Ticket with Card selected.
  2. The app calls Stripe Terminal to present payment on the reader.
  3. Customer taps/inserts — Stripe authorises instantly.
  4. Order saved as card payment automatically.
  5. Tip prompt on the reader can be enabled in Stripe Terminal settings.
  6. Stripe does not auto-print receipts — email/SMS receipt is sent by Stripe if configured. Use this app's "Print this ticket" button for a paper copy.

Testing: use Stripe's test mode key (sk_test_...) and a simulated reader — no hardware required. Docs: stripe.com/docs/terminal
